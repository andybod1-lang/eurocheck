# Landing Page Copy Prompts

## 1. Product Landing Page (Direct Response)

```
Write landing page copy for [PRODUCT_NAME].

Product: [BRIEF_DESCRIPTION]
Target audience: [WHO_NEEDS_THIS]
Primary keyword: [KEYWORD]
Main CTA: [CALL_TO_ACTION]

Generate:

### Above the Fold
- Headline: Clear value proposition (max 10 words)
- Subheadline: Explain what it does (max 20 words)
- CTA button text
- Social proof snippet

### Problem Section
- Paint the pain points (3-4 bullet points)
- Agitate: What happens if they don't solve it
- Transition to solution

### Solution Section
- How [PRODUCT] solves the problem
- Key features with benefits (feature → so you can → result)
- Format: 3-4 features with icons

### Social Proof Section
- Testimonial format (name, role, quote, result)
- Generate 3 realistic testimonials
- Stats or numbers if applicable

### How It Works
- 3 simple steps
- Each step: Title + 1 sentence

### Features Deep Dive
- 6 key features
- Each: Icon-sized title + 2-sentence benefit

### FAQ
- 5 common objections turned into Q&A
- Address pricing, trust, technical concerns

### Final CTA
- Urgency or scarcity element
- Risk reversal (guarantee)
- Button text + supporting copy

Tone: [TONE - confident/friendly/professional]
```

## 2. SaaS Landing Page

```
Create landing page copy for [SAAS_NAME] - [ONE_LINE_DESCRIPTION].

Target: [IDEAL_CUSTOMER_PROFILE]
Price point: [PRICING_MODEL]
Main competitor: [COMPETITOR]
Key differentiator: [WHAT_MAKES_US_DIFFERENT]

Sections needed:

1. HERO
- Headline focusing on outcome, not features
- Subhead explaining the how
- 2 CTAs (primary action + secondary)
- Quick benefit bullets (3)

2. LOGOS SECTION
- Text: "Trusted by [TYPE] at..."
- 5-6 company types

3. PROBLEM STATEMENT
- "If you're a [ROLE], you know..."
- 3 frustrations with status quo

4. SOLUTION INTRO
- "[PRODUCT] is different because..."
- Single-sentence positioning

5. FEATURE BLOCKS (3)
- Feature 1: [MAIN_FEATURE] → benefit
- Feature 2: [SECOND_FEATURE] → benefit
- Feature 3: [THIRD_FEATURE] → benefit

6. SOCIAL PROOF
- Testimonial with specific results
- Case study teaser

7. COMPARISON
- Why [PRODUCT] vs [COMPETITOR]
- 3 differentiators

8. PRICING PREVIEW
- Pricing philosophy
- "Starting at" or "Free to try"

9. FINAL CTA
- Headline: Create urgency
- Risk reversal
- Two button options
```

## 3. Free Tool Landing Page

```
Write copy for a free tool landing page.

Tool: [TOOL_NAME]
What it does: [DESCRIPTION]
Target keyword: "free [KEYWORD]"
Lead magnet goal: [WHAT_WE_CAPTURE - email, nothing, etc.]

Generate:

HEADLINE: "[Action verb] + [Benefit] — Free"
SUBHEAD: Explain what they'll get in 15 words

DEMO/PREVIEW SECTION
- Text introducing the tool
- What to expect

BENEFITS (3-4)
- Immediate value proposition
- Why this free tool
- No catches messaging

HOW TO USE
- 3 simple steps
- Time to complete

WHO IT'S FOR
- 3 use cases
- "Perfect for [ROLE] who..."

LEAD CAPTURE (if applicable)
- Value exchange copy
- What they get for email
- Privacy reassurance

ABOUT US
- Brief credibility statement
- Why we built this

RELATED RESOURCES
- Links to related content
- Upsell to paid product (soft)

SEO: Include [KEYWORD] naturally 3-4 times
```

## 4. Coming Soon / Waitlist Page

```
Create a coming soon landing page for [PRODUCT].

Launch date: [TIMEFRAME]
What it will do: [DESCRIPTION]
Target signup goal: [NUMBER]

Generate:

HEADLINE: Build anticipation + curiosity
SUBHEAD: What problem it solves

THE PROMISE
- 3 key benefits they'll get
- Why it's worth waiting for

EARLY ACCESS OFFER
- What waitlist members get (discount, early access, input)
- Exclusivity angle

EMAIL CAPTURE
- Input field label
- Button text (not just "Submit")
- Privacy note

SOCIAL PROOF
- How many signed up
- Notable people interested

TIMELINE/ROADMAP
- What's coming when
- Build excitement

ABOUT THE TEAM
- Brief credibility
- Contact for questions

SOCIAL SHARE
- Referral incentive if applicable
- Share buttons prompt
```

## 5. Comparison Landing Page

```
Write a "[PRODUCT] vs [COMPETITOR]" comparison landing page.

Our product: [PRODUCT_NAME]
Competitor: [COMPETITOR_NAME]
Target keyword: "[PRODUCT] vs [COMPETITOR]"

Generate:

HEADLINE: Fair comparison angle
SUBHEAD: "Here's how to choose..."

QUICK COMPARISON TABLE
| Feature | [PRODUCT] | [COMPETITOR] |
- 6-8 key features
- Honest comparison (don't trash competitor)

DETAILED COMPARISON
- Section for each major difference
- When [COMPETITOR] might be better
- When [PRODUCT] is the clear choice

USE CASE RECOMMENDATIONS
- "Choose [COMPETITOR] if you need..."
- "Choose [PRODUCT] if you need..."

SWITCHING SECTION (if applicable)
- How easy to switch
- Migration help offered

TESTIMONIALS
- From people who switched
- Specific comparisons they make

FINAL VERDICT
- Summary recommendation
- CTA to try [PRODUCT]

FAQ
- Comparison questions
- Migration questions
- Pricing comparison

Tone: Objective, helpful, confident but not arrogant
```

## 6. Feature Page

```
Create a landing page for [FEATURE_NAME] feature of [PRODUCT].

Feature: [DESCRIPTION]
Main benefit: [WHAT_IT_ENABLES]
Target keyword: "[PRODUCT] [FEATURE]"

Generate:

HEADLINE: Benefit-driven, includes feature name
SUBHEAD: How it works in one sentence

HERO VISUAL DESCRIPTION
- What screenshot/video should show
- Key UI elements to highlight

FEATURE OVERVIEW
- What it does (3 sentences)
- Why we built it
- Who it's for

KEY CAPABILITIES (4-5)
- Capability name
- What it enables
- Use case example

WORKFLOW SECTION
- Step-by-step how to use
- Visual flow description

INTEGRATION/COMPATIBILITY
- What it works with
- Technical requirements

CUSTOMER QUOTE
- Testimonial about this specific feature

RELATED FEATURES
- What else they might need
- Links to other feature pages

CTA SECTION
- Try this feature
- See it in action
- Talk to sales
```

## Tips for Landing Pages

1. **Focus on ONE conversion goal** per page
2. **Lead with benefits**, not features
3. **Use specific numbers** when possible
4. **Address objections** before the CTA
5. **Make the CTA obvious** and compelling
6. **Test headlines** — they're 80% of the battle
